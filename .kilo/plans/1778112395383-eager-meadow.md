# Plan: Create TEST.txt for wc Compatibility Testing

## Goal
Create a `TEST.txt` file with variable line lengths, word lengths, and character counts to verify the accuracy of a word-counting program against `wc` behavior.

## Details
- **File Path**: `TEST.txt`
- **Content Requirements**:
    - Non-offensive, professional, or pseudo-random text.
    - Nature-focused pseudo-text (similar to Lorem Ipsum but with nature themes).
    - Variations in:
        - Line lengths (short, medium, long).
        - Word lengths (single letters, long words).
        - Character counts (including whitespace and punctuation).
    - Structure: A "novel" take on Lorem Ipsum (e.g., tech-focused or nature-focused pseudo-text).
- **WC Compatibility Goals**:
    - Test line count (`-l`).
    - Test word count (`-w`).
    - Test byte/character count (`-c` / `-m`).

## Steps
1. Generate a structured set of text that ensures edge cases are covered:
    - Empty lines.
    - Lines with trailing whitespace.
    - Lines with only whitespace.
    - Very long words (no spaces).
    - Mixed punctuation.
2. Write the content to `TEST.txt` in the workspace root.
